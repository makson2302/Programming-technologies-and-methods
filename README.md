# Programming-technologies-and-methods
Программа для учёта измерений температуры с поддержкой графического интерфейса и пакетной обработки команд из текстового файла.
________________________________________
 О проекте
Проект разработан в рамках учебного задания и включает 4 последовательные итерации:
1.	Базовая версия – парсинг строки вида TemperatureMeasurement ГГГГ.ММ.ДД "место" значение, создание объекта, вывод в консоль.
2.	Графический интерфейс – таблица (Tkinter), добавление, редактирование, удаление записей, сохранение в файл data.txt, логирование ошибок.
3.	Добавление координат – ввод широты и долготы с обязательным знаком +/-, отображение в таблице с подписями (с.ш./ю.ш., в.д./з.д.).
4.	Финальная версия – удаление координат, добавление обработчика команд из внешнего файла (ADD, DELETE, EDIT, SORT, CLEAR), модульные тесты с покрытием кода ~88%.
В текущем репозитории представлена финальная версия программы.
________________________________________
 Функциональные возможности
•	Графический интерфейс на Tkinter:
o	Таблица со столбцами: Дата, Место, Значение (°C).
o	Добавление нового измерения через поля ввода.
o	Редактирование существующей записи (диалоговое окно).
o	Удаление выбранной записи.
•	Автосохранение в файл data.txt (формат: TemperatureMeasurement ГГГГ.ММ.ДД "место" значение).
•	Обработка командного файла (кнопка «Выполнить команды из файла»):
o	Поддерживаемые команды:
	ADD YYYY.MM.DD "location" value
	DELETE index
	EDIT index field new_value (field: date, location, value)
	SORT field [asc|desc] (field: date, location, value)
	CLEAR – удалить все записи.
o	Команды применяются к текущему списку измерений (загруженным + изменённым вручную).
o	Результат отображается в таблице и сохраняется в data.txt.
•	Логирование ошибок:
o	Некорректные строки в data.txt пропускаются, информация записывается в errors.log.
o	Ошибки выполнения команд показываются в диалоговом окне.
•	Модульные тесты (unittest) с покрытием ~88% (модель, файловое хранилище, парсер команд).

 Установка и запуск
Требования
•	Python 3.7 или выше (Tkinter входит в стандартную поставку).
Запуск
bash
git clone https://github.com/your-username/temperature-manager.git
cd temperature-manager
python temperature_program.py
Запуск тестов
bash
python -m unittest test_temperature.py -v
Оценка покрытия кода тестами
bash
pip install coverage
coverage run -m unittest test_temperature.py
coverage report -m
________________________________________
 Формат файлов
data.txt (основное хранилище)
Каждая строка – одно измерение:
text
TemperatureMeasurement 2023.10.05 "Laboratory" 23.5
TemperatureMeasurement 2024.01.15 "Kitchen" 19.8
commands.txt (пример файла с командами)
text
# Добавить измерение
ADD 2024.05.20 "Moscow" 18.5
# Удалить первое измерение
DELETE 0
# Отсортировать по дате
SORT date asc
# Очистить всё
CLEAR
errors.log
Лог ошибок парсинга data.txt и ошибок выполнения команд.
________________________________________
Тестирование
Модульные тесты покрывают:
•	Класс TemperatureMeasurement (парсинг, сериализация).
•	Класс FileStorage (чтение/запись, пропуск некорректных строк).
•	Класс CommandProcessor (все команды: ADD, DELETE, EDIT, SORT, CLEAR, обработка ошибок).
GUI-часть не покрыта тестами (задание не требовало), но вся бизнес-логика протестирована.
